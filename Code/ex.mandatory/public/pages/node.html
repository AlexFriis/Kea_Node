Node

What is node and how do we use it?

Node.js is an asynchronous event-driven JavaScript runtime, which is designed to build scalable network applications.
This can be done with various frameworks and in this project Express has been used.


Assuming we start out with an empty project we first create a file called app.js
In this file we specify configurations for the application.
This could be to initialize express, define serverport or create our endpoints.

To run a node application open a terminal of choice and enter the following: node applicationName.js 


                  //Example of Express from expressjs.com
                  //Initialize express in our app
                const express = require('express');
                const app = express();
                //Define port we want to run the application on
                const port = 3000;
                
                
                app.get('/', (req, res) => {
                  res.send('Hello World!')
                })
                
                //Start the server on specified port, console log the port number
                app.listen(port, () => {
                  console.log(`Example app listening on port ${port}`)
                })

Nodemon

Nodemon is a tool only used in production.
Many IDE's allow users the functionality of reloading your application as changes are made.
This functionality also exists for node, in the form of nodemon.
Nodemon wraps the node app, watches the filesystem and automatically restarts the process.
It's a critical tool used in development to save time by not having to input "node app.js" to run your application manually every time changes are made. 


                  //How to install nodemon

                  //Can be installed through git clone or by using NPM.

                  //-g installs nodemon globally
                  npm install -g nodemon

                  //--save-dev
                  //With a local installation nodemon will not be available in the system path and therefore cant be used directly in the command line.
                  //The local installation can be run by calling it from within an npm script
                  npm install --save-dev nodemon

                  //Start a nodemon app
                  nodemon app.js

Package.json

Diving further into node and its uses we'll take a look at the package.json file.
package.json is the center of a node project. It records important metadata about a project which is required before publishing to npm.
It also defines functional attributes of a project that npm uses to install dependencies, run scripts and identify the entrypoint to the package.

Dependencies are best managed using npm commands such as npm install, npm uninstall and npm update to keep the node_modules in sync.
By manually adding dependencies without using terminal commands we still have to run npm install before the dependency is actually added to the project. 


                  //Package.json for this project

                  //dependencies are written in json format.

                  {
                    "dependencies": {
                        "express": "4.17.2"
                    }
                }
